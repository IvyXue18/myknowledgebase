如果你正在经营自己的产品、服务，希望通过内容获客，却苦于没有流量、转化效率低、内容生产不持续...也许该停下来重构一下整个内容制作的流程。

以前写种草文案，我常常会以自己有什么出发，比如我的产品有什么功能、适合什么人、有什么卖点，然后再去成一篇文章。

但后来我发现，这样写出来的内容很难打动人。

因为它还是产品视角，不是用户视角。

真正高转化的内容，顺序应该反过来：

**不是去问 “我的产品有什么”，而是先看用户已经在关心什么、卡在哪里、有哪些需求，再去用自己的产品和服务去做行动承接。**

而评论区，就是一个非常好的用户需求现场。

用户的抱怨、犹豫、求推荐、求教程、求链接，其实都在告诉你：

```
他遇到了什么问题；
他为什么还没行动；
他想要什么结果；
他需要一个什么样的解决方案。
```

最近我用 **媒讯助手 + 飞书多维表格 + AI** 搭了一套内容获客流程：

```
1️⃣ 采集热门内容和评论
2️⃣ 提取用户痛点和真实场景
3️⃣ 生成选题、标题和内容大纲
4️⃣ 沉淀成内容资产库
```

跑下来最大的感受是： **内容不再是凭灵感硬想出来的，而是从用户真实需求里长出来的。**

**今天手把手教你用飞书媒讯助手，搭建出一套内容获客工作流，适合卡在 “不知道写什么”“写了没人转化”的老板朋友们，然后你就** 可以得到：

🌟 一个持续产生洞察的用户需求库；

🌟 一批更容易引起用户共鸣和信任的内容选题方向；

🌟 一套完全可以自己跑起来的内容获客工作流

从此不仅告别内容难产的问题，还能让内容真正服务自己的服务与产品，提高转化率。

02

从评论挖掘痛点需求

评论采回来以后，接下来就是对数据的清洗和过滤，优先看这 3 个方面。

第一类是 **高赞评论** 。

一条评论被很多人点赞，说明它不只是一个人的感受，而是很多人都有类似共鸣；这类评论很适合变成下一篇内容的选题。

第二类是 **抱怨评论** 。

用户愿意抱怨，说明他有期待，但现有方案没有满足他。

很多新的内容机会，其实就藏在用户的抱怨里。

第三类是 **求推荐、求教程、求链接、求解决方案的评论** 。

这类评论尤其重要，因为它说明用户已经不只是“看看”，而是有行动意愿。

如果他已经在问：有没有办法？有没有推荐？哪里能找到？具体怎么做？

这类评论往往比普通点赞收藏更接近转化。

这一步可以直接交给 AI 来做分析，先让它理解用户。

这里 Prompt 可以参考 ⬇️

```
请你根据下面这些用户评论，帮我提取：1. 用户真正想要的结果2. 用户正在经历的问题3. 用户出现问题的具体场景4. 用户的情绪和顾虑5. 可以转化成内容选题的角度
要求：- 不要泛泛总结- 尽量保留用户原话- 每个洞察都要对应至少 1 条评论证据
这里有两个细节要注意。
```

第一， 尽量让 AI 总结出比较具体完整的场景，包括在什么场景下受到了什么样的限制，想要达到什么样的效果，做更详细的场景还原分析。

![图片](https://mmbiz.qpic.cn/mmbiz_png/QVW0eI0tJCPIxZUSmibzdReGyMVCkLhIicMkpfBOiaWBsQ8gfQiccRYME7bk0zibSCwyaTUZwJ198wfQP6p8bmCPlL2vw4sviayicUk3dK9J7LtKVU/640?wx_fmt=png&from=appmsg#imgIndex=5)

第二，要尽量保留用户原话。用户原话里有很多内容表达的线索。你后面写标题、写开头、写场景时，最有用的往往不是 AI 总结出来的抽象词，而是用户自己说出来的那句话，这也会让 AI 生成的内容更有人味。

03

基于用户反馈批量生成获客内容

等 AI 把用户痛点、场景、情绪和顾虑提取出来以后，再让它生成内容选题。

因为有了足够丰富的对于用户需求痛点的理解，这时候 AI 生成出来的内容，就不是凭空想出来的了，能更容易戳中用户痛点。

内容选题的生成可参考这个 Prompt：

```
请基于上面的用户痛点和场景，帮我生成 20 个适合小红书 / 公众号 / 短视频的内容选题。每个选题包含：1. 标题2. 对应用户痛点3. 目标人群4. 内容角度5. 文末可以引导的下一步动作如果想继续往下写文章大纲，会再加一步：
```
```
请从上面的选题里，挑出最有转化潜力的 5 个，并为每个生成内容大纲。要求：- 开头直接命中用户痛点- 中间用用户场景展开- 不要写成产品说明书- 最后自然引出我的产品 / 服务 / 工具这一步做完以后，你会发现，内容生产会变得非常具体。
```

以前你可能是在想：今天我到底写什么？

现在你看到的是：

```
用户正在经历什么问题；
他为什么会卡住；
他想要什么结果；
我可以用什么角度帮他往前走一步；
最后可以自然引导他做什么动作。
```

你会发现，内容生产从未如此容易，不再是靠灵感写内容，而是从用户需求里长出内容，解答用户的困惑，满足他们的需求，也更容易植入你的产品、服务作为后端转化承接。

通过这个方式，可以高效率的产出高转化的内容选题 ⬇️

下面是基于用户的评论，出的选题 + 文案，再根据文案一键直出的 AI 数字人带货视频，大家也可以感受一下效果。

选题直接从评论区里面挖掘出来了一个卡点：删了相册也没有优化空间，再基于这个卡点，给出新的解决方案，最后自然的引出产品。

这种方式，内容生产效率不仅大大提升，也会让内容与用户在关注的话题相关。

04

沉淀成内容资产库

每次采集完评论、分析完痛点、生成完选题以后，我会把这些东西沉淀下来。

最简单的方法就是放进飞书。

可以分成几列：

```
关键词库差评库评论原贴库用户痛点场景档案库内容选题库这样做的好处是，每一次的积累都是让你的知识库越来越强大，下一次写内容时，也不是每天重新想选题，而是从用户需求库里取内容。
```

这才是我理解的内容资产。

不是今天发一篇，明天再重新想一篇。

而是每一次评论采集、每一次内容分析，都会变成后面可以复用的素材。

把产品的卖点和用户的买点连接起来后，前者是在说“我有什么功能”，后者是在说“我理解你正在经历什么”。

种草内容也不会再干巴，是在说用户正在经历的问题。

让AI 提效的内容工厂，也能达到有效的转化力，这才是这套流程很有价值的地方。

如果你也想试试这套流程，可以先从最简单的闭环流程跑起来。

你只需要做三件事：

1️⃣ 选一个和你业务相关的关键词；

2️⃣ 用飞书媒讯助手采集 20-50 条热门内容和评论；

3️⃣ 用上面的 Prompt 提取用户痛点和选题，并基于用户的洞察写出内容。

先跑一次，你就能明显感觉到： 内容不是从脑子里硬想出来的，而是从用户真实需求里长出来的。
