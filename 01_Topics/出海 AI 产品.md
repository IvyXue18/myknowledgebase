## 核心问题

普通人 / 新手如何从 0 到 1 做出一个能赚钱的出海 AI 产品(独立网站 / SaaS),把"找需求—上线—获取流量—转化付费"这条链路完整跑通,而不是停留在做产品的幻觉里。

## 关键观点

- 营收 = 流量 × 转化率 × 客单价 × 复购。新手期最该追求的不是单个项目多完美,而是先用一个最小功能把这个闭环跑通一次,再针对薄弱环节(流量/转化率/定价)逐项优化。
- 万事问 AI,决定天花板的不是 AI 有多强,而是你和 AI 的关系有多亲密——把 AI 当架构师、产品经理、定价顾问、心力教练,而不只是写代码工具。
- 新词站能快速拿到正反馈(新人最缺的就是正反馈),但热度会衰减;真正的复利在老词 + 产品体验 + SEO 的长期积累。新词站可以作为冷启动流量,导流给主打长期需求的品牌词站。
- 新手该"听话照做",先抄成熟竞品的定价结构(年付价格锚点默认展示),而不是凭直觉简化,直觉往往是错的。
- 任何新产品都有 90% 以上的失败率,新手期该增加的是分母(项目数量),而不是死磨一个项目到完美再上线。
- 决定项目能不能撑过没有正反馈的阶段的,往往是心态和现金流是否稳定,而不是技术能力。

## 方法框架

### 找需求 / 挖新词
- 社媒(X/Twitter 头部 AI 账号)+ Google Trends 验证:看到新模型/新概念先去 Trends 确认是不是真新词、有没有流量。
- Huggingface Spaces 新模型 + Google Trends:盯着最近更新的开源模型,同样用 Trends 验证。
- 榜单(toolify、TAAFT、Traffic.cv)+ Similarweb:分析竞品落地页(尤其是 huggingface 这类聚合站的内页)流量趋势,反推还在上升的关键词,再用 Google 搜索验证竞争强度,找"软柿子"(排名靠前但产品很单薄的站)。
- 日常记录习惯(参考 Pieter Levels):强制自己每天记录至少 3 个需求,把刷视频、刷社媒时的痛点随时记下来。
- 判断需求真伪:能否一句话说清"什么人,在什么场景下,愿意花多少钱,解决什么问题";能否 eat your own dog food;假设这个产品出自一个你讨厌的人,你还会不会付费——能,大概率是真需求。

### 上线与 SEO
- MVP 原则:只做一个核心功能,2 周内上线第一版,先验证再打磨;功能列表按真伪需求标准逐项划掉,留下和初衷最贴的那个。
- SEO 三件事:关键词(一词一页,长尾词包围核心词,即"农村包围城市")、外链 / 内链(内链帮 Google 理解站点结构,外链要坚持发)、产品体验(移动端性能用 PageSpeed 检测,Google 对移动体验权重很高)。
- 常见技术坑(踩中会导致排名归零):多语言上线后 `/ja` 与 `/ja/` 被当成重复页面;复用有流量站点的模板导致其他语言页面在不同域名间重复;sitemap 漏了 `hreflang="x-default"`;http 没有 301 到 https。修复(robots.txt disallow + GSC 删除收录 + 加外链)后约一周排名可恢复。
- AdSense 被拒"低价值内容"的常见解法:补齐 /about /contact /blog 等页面,证明站点不是单页空壳。

### 流量获取——"流量四板斧"
SEO、平台广告投流、红人传播、自己发帖灌水,核心还是围着当下的热点做,不要分散打杂毛。
- Reddit:养号攒 Karma,发链接前按"评论爆款帖→发不带链接的求助帖→小号互动带链接→正式发链接帖(最好先问群主)"逐步试探;先用纯文本链接,帖子火了再改成超链接。
- YouTube:腰部 AI 博主的评测/联盟返佣能带来持续数月的长尾订单,效果不输 Reddit。
- 独立站提交:v2ex、IndieHackers、Product Hunt、Hacker News、BetaList 等技术/产品社区,以及 Toolify/Taaft 等付费导航站(约 99 美金级别)可以快速验证需求。
- Google Ads 细节:开号时选的国家后续不可改;留意新客赠金档位;统计代码(GA/Tag Manager)要尽早装好,否则广告系统拿不到转化数据;看 14 天 ROAS 而不是单日;可以找平台官方顾问免费咨询。

### 定价与转化
- 默认展示年付价格锚点,年付至少打 5 折(5 折是质变临界点)并同步提高年费权益,实测能把年订阅占比拉高 30% 以上;只要算清 API 成本,基本不会因为打折而亏。
- 免费额度按用量给(每天 N 次 / 注册送 N 积分),而不是按天数给试用期,既能让用户体验到核心价值,又不会白嫖到满足。
- 转化辅助手段:取消订阅时的"挽留优惠"弹窗、限时折扣制造 FOMO、首页放真实用户评价 / 实时使用人数(从众心理)、邮件召回(未续费/留存好但未付费用户),以及对高价值用户做 Upsell。
- 数据上判断问题出在哪:Plausible 看 /pricing 页访问量 vs 成交量——访问多成交少多半是价格问题,UV 高但点 /pricing 少多半是体验或人群不精准;Clarity 录屏直接定位具体的 UX bug(如多语言硬编码错误)。

### 产品 / AI 协作模式
- 分工:ChatGPT(gpt 5 thinking)当架构师 + 代码审查,Claude Code / Codex 当执行的高级程序员,复杂逻辑(支付、积分、缓存)交给架构师把关。
- 定价方案直接让 AI 算:把 API 成本告诉它,它能给出保证不亏钱、毛利 75%+ 的定价结构。
- 数据栈组合:GA/GSC(基础流量)+ Plausible(转化与国家分布,有 API 可二次开发)+ Clarity(行为录屏)+ Metabase 连接业务数据库(neon)做可视化大屏,n8n 负责把 Plausible 数据同步进数据库。

## 典型案例

- **italian brainrot names**:通过 toolify(榜单)→ Similarweb(分析竞品落地页关键词)→ Google Trends(验证流量是否新爆发)的链式方法,找到一个一个月内才出现的新词,验证后注册域名上线。
- **nanobanana.ai / nano-banana.pro**:作为"学习对象"案例,展示新词站如何用年付锚点定价、双支付方式、主页+独立功能内页(而非堆在一个页面里)的结构做 SEO 和转化。
- **王马扎的品牌词 AI 工具站**:用先跑起来的新词站导流做冷启动,上线 5 天出第一单 327 刀年订阅,14 天破千刀,印证"小流量(日 UV 100+)+ 高客单订阅"也能跑通闭环。
- **小耳朵的出海项目**:Reddit + YouTube 免费流量起步,叠加 Google Ads 放大,配合年付折扣 + 权益加码,把月营收做到接近 4 万刀级别。

## 反例和争议

- 过度完美主义导致项目烂尾:王马扎自己有个工具站打磨了一个多月细节,至今没上线;两人都提到见过同行因为"想做到完美才上线"而错过窗口期。
- 直接照抄竞品定价、不核算自己的 API 成本,容易出单也亏钱——必须先算毛利再定价。
- 押注新词热度衰退快,新词站第一个月流量高、后续持续下跌是常态,长期看产品体验 + 老老实实做 SEO 才是更稳的路径。
- 一次性上线大量低质量页面(尤其 AI 生成内容)、一上来就做多语言、一个页面塞多个关键词,是新手最容易踩的几个 SEO 坑。
